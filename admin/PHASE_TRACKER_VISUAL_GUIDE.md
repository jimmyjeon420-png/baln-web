## 진행률 계산 로직

```
총 작업 수 = 5개
완료된 작업 = 2개

진행률 = (2 / 5) × 100 = 40%
진행률 바 너비 = 40%
카운터 = "2/5"
```

**상태별 우선순위:**
1. 블로킹 있음 → "⚠️ N개 블로킹 중"
2. 진행 중 있음 → "🔄 N개 진행 중"
3. 모두 완료 → "✅ 완료"
4. 기본 → "진행률 N%"

## 실제 사용 예시

### 시나리오 1: 작업 시작

**현재 상태:**
```
⬜ "오늘" 탭 리디자인 🔴 긴급
```

**작업자가 아이콘 클릭:**
```
🔄 "오늘" 탭 리디자인 🔴 긴급
```

**결과:**
- Phase 9 상태: "🔄 1개 진행 중"
- 진행률: 변동 없음 (아직 완료 아님)

### 시나리오 2: 작업 완료

**현재 상태:**
```
🔄 "오늘" 탭 리디자인 🔴 긴급
진행률 40%  (2/5)
```

**작업자가 아이콘 클릭:**
```
✅ "오늘" 탭 리디자인 🔴 긴급
진행률 60%  (3/5)
```

**결과:**
- 진행률 바: 40% → 60%
- 카운터: 2/5 → 3/5
- 작업 색상: 노란색 → 녹색

## 성능 최적화

- **캐싱**: phaseTasksCache로 데이터 메모리 저장
- **레이지 로드**: Phase 클릭 시에만 상세 목록 렌더링
- **트랜지션**: CSS transition으로 부드러운 애니메이션 (0.3s)
- **이벤트 위임**: 개별 작업마다 이벤트 리스너 대신 HTML onclick 사용

## 접근성 (a11y)

- **클릭 가능 표시**: cursor: pointer
- **사용자 선택 방지**: user-select: none (아이콘 텍스트 선택 방지)
- **색상 대비**: WCAG AA 기준 준수
- **키보드 네비게이션**: 추후 추가 예정 (Tab, Enter)
